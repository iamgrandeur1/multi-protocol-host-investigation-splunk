Project 13 – Multi-Protocol Host Investigation & Threat Hunting Using Splunk

In this project, I conducted a host-centric threat hunting investigation using Splunk Enterprise and Zeek network telemetry. Rather than focusing on a single protocol, I correlated activity across DNS, HTTP, FTP, and SSH logs to build behavioral profiles of internal systems.

The investigation included:
• Identifying the most active internal hosts
• Correlating activity across multiple Zeek log sources
• Analyzing DNS request patterns
• Investigating HTTP communication
• Examining SSH activity
• Profiling hosts based on protocol usage

This project demonstrates how multi-protocol analysis provides greater visibility into network behavior and supports effective threat hunting within a Security Operations Center (SOC).
